# Integration Scorecard

## Prompt

I'm CVP of Corporate & Business Development at Ripple. We closed four acquisitions in 2025 — Hidden Road, GTreasury, Rail, and Palisade — and each is at a different stage of integration. I need a detailed, board-ready integration scorecard that goes deeper than "on track" or "behind" — it should show exactly where each acquisition stands across every integration dimension, what's working, what's not, and what needs executive intervention.

Using my connected sources (Gmail, Slack, Google Drive) — or the attached `integration_status.csv` from the primary exercise as fallback — build me a per-acquisition Integration Scorecard.

### For Each Acquisition, Generate:

**1. Executive Summary**
2-3 sentences on overall integration health. Honest, not spin. Where are we vs. the Day 1/Day 100/Day 365 plan?

**2. Dimension Scorecard**
RAG status and detail for each dimension:

- **Technology:** Platform integration, API connectivity, data migration, infrastructure consolidation. What's merged, what's running in parallel, what's blocked.
- **People:** Retention rates, cultural integration, org design, key person risk. Flag any departures or retention concerns.
- **Customers:** Customer communication, churn risk, cross-sell activation, NPS/satisfaction changes since acquisition. Are customers seeing value from the combination?
- **Product Synergies:** Cross-product features shipped or in development, revenue from synergy-driven deals, pipeline of synergy opportunities. Are the deal thesis synergies materializing?
- **Financial:** Integration cost vs. budget, revenue tracking vs. acquisition model, path to target margins.

**3. Key Decisions Needed**
For each acquisition, what decisions are pending that could accelerate or unblock integration.

**4. 90-Day Outlook**
What should be true 90 days from now if integration goes well? What are the risks to getting there?

### Output Format
One page per acquisition as a structured HTML document. Use a consistent template so they can be compared side-by-side. Include a summary comparison table at the top.

## Stretch Goals

1. **Integration lessons learned** — Ask Claude to identify patterns across the four integrations: what's working consistently, what's failing consistently, and what we should do differently for the next acquisition.
2. **Resource allocation optimizer** — Show how integration resources (engineering, legal, compliance) are distributed across the four acquisitions and recommend rebalancing.
